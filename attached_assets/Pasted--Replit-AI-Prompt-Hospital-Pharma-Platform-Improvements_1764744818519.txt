# Replit AI Prompt - Hospital Pharma Platform Improvements

## Overview of Changes
Implement major workflow improvements to the Hospital Pharma Procurement Platform including a new label-based navigation system, enhanced visualizations, RCA/recommendations screen, and vendor forecast reporting.

---

## 1. NEW FIRST SCREEN - Label Dashboard (Landing Page After Login)

### Requirements:
Create a new landing page that displays procurement labels as the main navigation:

### Label Categories to Display:
1. **Over-Stocking** - Products with excess inventory
2. **Under-Stocking** - Products below optimal levels
3. **Expiry Risk** - Products nearing expiry (< 30 days)
4. **Wastage** - Products wasted due to expiry or damage
5. **Dead Stock** - Products with no movement (> 90 days)
6. **Stockout Risk** - Products at critical low levels
7. **High Velocity** - Fast-moving products requiring attention
8. **Slow Moving** - Products with declining consumption
9. **Emergency Shortage** - Critical medicines running low
10. **Cost Overrun** - Products exceeding budget allocation

### Label Card Design:
Each label should be displayed as a card showing:
- **Label Name** (large, bold text)
- **Affected Products Count** (e.g., "23 products")
- **Total Value/Quantity** affected
- **Status Icon** (Alert/Warning/Check icon)
- **Color-coded Border and Background**:
  - **Green**: All products normal (0-5 affected items)
  - **Yellow**: Moderate issues (6-15 affected items)
  - **Red**: Critical issues (16+ affected items)
- **Trend Indicator**: Arrow showing if situation is improving/worsening
- **Clickable** to navigate to detailed view

### Layout:
- Grid layout: 3-4 columns on desktop, 2 on tablet, 1 on mobile
- Animated cards with hover effects (lift and shadow increase)
- Each card pulses gently if status is Red (critical)
- Sort cards by severity (Red first, then Yellow, then Green)

### Top Section:
- Show "Procurement Health Overview" heading
- Display overall health score (e.g., "78/100" with color indicator)
- Show last updated timestamp
- Quick stats: Total SKUs, Total Value, Active Alerts

### Synthetic Data Requirements:
Generate realistic data for each label:
- Product counts per label (mix of red/yellow/green states)
- Total monetary value affected
- Trend data (7-day history showing improvement/decline)
- Severity distribution across labels
